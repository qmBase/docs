Innerhalb von qmBase werden bei vielen Ereignissen Benachrichtigungen ausgelöst, so dass Personen, die informiert bleiben sollen, dies auch tun.
Die Benachrichtigungen werden pro Ereignis als E-Mail verschickt.
Wenn die Benachrichtigungen eine Aktion erfordern, enthält die Benachrichtigung einen Hinweis und einen Link, was zu tun ist.

Jede Benachrichtigung wird zeitgleich nur an eine Person verschickt.
Zusätzlich gibt es E-Mail Zusammenfassungen, die regelmäßig verschickt werden und eine Übersicht der aktuellen Benachrichtigungen auf der Startseite.

### Wer erhält Benachrichtigungen?

Automatische Benachrichtigungen erhält jeder Nutzer der Software. Nutzer der Software sind diejenigen, die bereits über einen Zugang zu <ProductDisplayName/> haben.
D.h. Personen, die nur in der App Mitarbeiter angelegt sind, erhalten **keine** Benachrichtigung.
Personen, die keine Benachrichtigung erhalten können, werden an vielen Stellen mit der im Screenshot dargestellten Glocke dargestellt.
![](https://caqadmin.blob.core.windows.net/public-screenshots/manual-screenshots/Screenshot%202022-02-14%20102048_avatarNoNotification.png)

#### Werden Benachrichtigungen nachträglich verschickt, wenn z.B. ein Kollege sich erst später registriert?

Nein, Benachrichtigungen werden nicht nachträglich verschickt. Aufgaben, die einem Kollegen zugeordnet sind, werden aber sichtbar, sobald der Kollege sich als neuer Nutzer registriert.
Zur Erklärung der nachträglichen Zuordnung von Nutzern und Mitarbeitern finden Sie [hier weitere Informationen](/docs/account/account-login#verknüpfung-von-account-und-personen).

## E-Mail Erinnerung und Zusammenfassungen

Wir verschicken wöchentlich Zusammenfassungen der anstehenden Ereignisse der kommenden 7 Tage. Diese werden in Form einer Tabelle zusammengefasst und pro App zusammengefasst verschickt.
Für die einzelnen Apps werden folgende Informationen verschickt:

| App                     | Inhalte                                                                          |
| ----------------------- | -------------------------------------------------------------------------------- |
| Auditmanagement         | Audits als Verantwortlicher, Audits als Teilnehmer                               |
| Blog                    | Dokumente mit abgelaufenem Überwachungsintervall                                 |
| Dokumententenmanagement | Dokumente mit abgelaufenem Überwachungsintervall                                 |
| Instandhaltung          | Anstehende Prüfungen von Inventargegenständen, für die man verantwortlich ist    |
| Ideenmanagement         | Ausstehende Gutachten als Gutachter einer Idee                                   |
| Projekte & Maßnahmen    | Projekte als Verantwortlicher, Maßnahmen als Verantwortlicher                    |
| Reklamationsmanagement  | Reklamationen als Verantwortlicher                                               |
| Risiken & Chancen       | Risiken und Chancen als Verantwortlicher                                         |
| Schulungsmanagement     | Anstehende Schulungen als Verantwortlicher, anstehende Schulungen als Teilnehmer |
| Wiki                    | Dokumente mit abgelaufenem Überwachungsintervall                                 |
| Zielmanagement          | Ziele als Verantwortlicher                                                       |

## Allgemeine Benachrichtigungen

Benachrichtigungen für Ereignisse, die an verschiedenen Stellen der Software auftreten werden im folgenden Beschrieben:

| Aktion             | Empfänger                    | Beschreibung |
| ------------------ | ---------------------------- | ------------ |
| Kommentar erstellt | Verantwortlicher / Ersteller |
| Datei angefügt     | Verantwortlicher / Ersteller |

## Design der Benachrichtigungen

Der Aufbau des Betreffs ist wie folgt: **Schweregrad** > **App** > **Zweck** > **#{ID} {Titel des zu Grunde liegenden Elements}**. Im Inhalt der Nachricht werden, falls notwendig, weitere Details beschrieben und ein Link zu dem zu Grunde liegenden Eintrag hinterlegt.

| Variable    | Beispiel                                                                              |
| ----------- | ------------------------------------------------------------------------------------- |
| Schweregrad | Info, Handlung erforderlich                                                           |
| App         | Name der App z.B. Ideenmanagement                                                     |
| Zweck       | Was soll durch die Benachrichtigung erreicht werden z.B. Idee muss freigegeben werden |
