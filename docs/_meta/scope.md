# Documentation Scope

Included:
- How to use product features
- Configuration guidance
- Conceptual explanations tied to product behavior
- Examples aligned with ISO 9001 terminology
- Common mistakes and how to avoid them
- Role-specific instructions

Excluded:
- Legal advice
- Certification guarantees
- Organization-specific process design
- Custom implementation consulting
- Internal architecture or source code details

The documentation explains **how the software behaves**, not how users must design their quality system.
